#### [E] Раздел 13 (Параллелизм) — Send+Sync гарантии, nested par_for_each, ссылка на Rhai

**Изменения:**
1. После 13.1 добавить разъяснение **почему** параллелизм безопасен:
   - Shared borrow на уровне архетипов — Rayon `scope` гарантирует отсутствие data race
   - Структурные изменения deferred — не мешают параллельному чтению
   - `par_for_each` не может быть вложен (Rayon запаникует)
2. В конце раздела 13 добавить подраздел **13.3 Ограничения**:
   - Nested `par_for_each` внутри `par_for_each` — паника
   - `par_for_each` + структурные изменения (insert/remove) из callback — deferred через Commands
   - Rhai-скриптинг однопоточен — не использовать в ParSystem (перекрёстная ссылка на раздел 17)
3. В раздел 17 (Rhai) добавить предупреждение: «Rhai работает только в однопоточном контексте — не вызывайте engine.run() внутри ParSystem или par_for_each»
